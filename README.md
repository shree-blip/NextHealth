# NextGen Healthcare Marketing

A full‑stack Next.js 15 application with TailwindCSS and a self‑hosted AdminJS backend for managing blog content.

---

## Getting Started

### Prerequisites
- Node.js (v18+ recommended)

### 1. Install dependencies

```bash
npm install
# or yarn / pnpm install
```

### 2. Development scripts

| Script | Description |
|--------|-------------|
| `npm run dev` | Start Next.js frontend on port 3000 |
| `npm run backend` | Compile & run Express/AdminJS on port 3001 |
| `npm run dev:all` | Run frontend + backend concurrently |
| `npm run db:push` | Push Prisma schema to SQLite database |
| `npm run prisma:gen` | Regenerate Prisma client |


---

## Blog & Admin Panel Setup

### Prisma models
The schema is defined in `prisma/schema.prisma` with `Post`, `Author`, `Category`, `Tag`, and `Comment` models. Posts now support relationships to categories, tags, related posts, and comments. Running `npm run db:push` will update the SQLite database schema.

### Seeding data
You can seed the database manually or by adding a seed script and running `npx prisma db seed --preview-feature`.

### Testing the blog
1. Insert a post using the Prisma client or AdminJS dashboard. You can now also add categories, tags and even comments via AdminJS (comments are created unapproved by default).
2. Visit `http://localhost:3000/blog` to see the index page. Each post listing shows title, excerpt and publish date.
3. Click a post to view it at `/blog/[slug]`. On the post you will see:
   - cover image (if any)
   - categories and tags
   - social share buttons (Twitter/LinkedIn)
   - an autogenerated table of contents
   - related posts based on shared categories/tags
   - a placeholder comments section that will pull from the new `Comment` model once an API is implemented.

### ISR and revalidation
Each blog route exports `export const revalidate = 300;` so pages regenerate every 5 minutes. When you update content via AdminJS, it will become live automatically within that window.

### AdminJS
- Start the backend with `npm run backend` (or `npm run dev:all`).
- Visit `http://localhost:3001/admin` to manage content.
- No authentication is configured yet; you should add middleware or integrate with NextAuth before deploying.

## Deployment Notes
- Serve the admin on a protected subdomain or behind VPN.
- Add proper environment variables (`ADMIN_PORT`, database path, auth secrets).
- Generate sitemaps/robots for SEO.
- Use webhooks or `revalidatePath` functions to force regeneration when needed.
